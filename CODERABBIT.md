# CodeRabbit 리뷰 가이드

## 브랜치 네이밍 컨벤션

CodeRabbit은 PR의 브랜치 이름 prefix를 기준으로 리뷰 포커스를 다르게 적용합니다.

| 브랜치 prefix | 의미           | 리뷰 포커스                             |
| ------------- | -------------- | --------------------------------------- |
| `feat/`       | 새로운 기능    | 기능 완성도, 타입 안전성, 컴포넌트 구조 |
| `fix/`        | 버그 수정      | 수정 범위, 사이드 이펙트, 재발 방지     |
| `refactor/`   | 리팩토링       | 기능 동일성, 불필요한 변경 여부         |
| `chore/`      | 빌드·설정 변경 | 설정 정합성, 불필요한 파일 포함 여부    |
| `style/`      | 코드 포맷      | 로직 변경 없음 확인                     |
| `docs/`       | 문서 변경      | 내용 정확성                             |

## 브랜치별 자동 리뷰 규칙

### `feat/*`

- `'use client'` 남용 여부 확인
- 새 컴포넌트의 named export / props interface 정의 확인
- API 호출 시 `apiClient` 사용 여부 확인
- 모바일 퍼스트 Tailwind 클래스 순서 확인

### `fix/*`

- 수정한 로직 외 불필요한 변경 포함 여부
- 동일 버그가 다른 위치에서도 발생 가능한지 확인
- 에러 처리(`ApiError`) 누락 여부

### `refactor/*`

- 외부 동작 변경 없이 내부 구조만 개선되었는지 확인
- 타입 정의 변경 시 영향 범위 확인

### `chore/*`

- `.env` 등 민감 정보 포함 여부
- `package.json` 버전 변경의 의도성 확인

## 리뷰 제외 경로

```
**/*.lock
**/node_modules/**
**/.next/**
**/dist/**
```

## 설정 파일

프로젝트 루트의 `.coderabbit.yaml` 에서 전체 설정을 관리합니다.

- 리뷰 언어: 한국어(`ko-KR`)
- 자동 리뷰 대상: `main` 브랜치로의 PR
- 드래프트 PR: 리뷰 제외
